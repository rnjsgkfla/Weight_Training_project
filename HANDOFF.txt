========================================================================
스쿼트 자세 피드백 프로젝트 — 인수인계 문서
========================================================================
다른 컴퓨터/다른 Claude 세션에서 이어서 작업하기 위한 문서.
이 문서만 읽으면 지금까지의 맥락·결정사항·남은 단계를 파악할 수 있다.

GitHub: https://github.com/rnjsgkfla/Weight_Training_project (Public)


------------------------------------------------------------------------
1. 프로젝트 목표
------------------------------------------------------------------------
스쿼트 기준(표준) 영상과 사용자 영상을 비교해 자세 피드백을 주는 프로그램.
- 입력: 사용자의 정면 / 측면 스쿼트 영상 2개
- 기준 영상: data/raw/squat_front_raw.mp4, data/raw/squat_side_raw.mp4
            (640x360, 24fps)
- 최종 목표: "무릎이 안쪽으로 모입니다", "엉덩이가 덜 내려갔습니다" 같은
            항목별 피드백 출력


------------------------------------------------------------------------
2. 지금까지 완료된 것 (파이프라인 1~4단계)
------------------------------------------------------------------------
raw 영상
  -> keypoint_extractor.py  : MediaPipe Pose 로 33관절 추출 -> CSV + 뼈대 영상
  -> smooth_landmarks.py    : Savitzky-Golay 로 좌표 스무딩
  -> normalize_landmarks.py : 골반중심 원점 + 상체길이 스케일 + 종횡비보정
                              + 측면 좌우통일
  === 여기까지 완료 ===================================================
  -> [남은 단계] 특징 추출 -> rep 최저점 검출 -> DTW -> 임계값 판정 -> 피드백

각 스크립트는 __main__ 에서 정면·측면 2개를 처리하고,
재사용 가능한 함수로 분리돼 있다.


------------------------------------------------------------------------
3. 환경 세팅 (새 컴퓨터에서)
------------------------------------------------------------------------
  python3.11 -m venv venv
  ./venv/bin/pip install -r requirements.txt

- Python 3.11 사용.
- 주의: mediapipe 는 반드시 0.10.14 로 고정(requirements 에 명시됨).
  최신 빌드(0.10.35 등)는 레거시 mp.solutions.pose API 가 빠지고 Tasks API 만
  남아 현재 코드가 동작하지 않는다.
- 검증용으로 matplotlib 을 쓸 수 있으나 프로젝트 코드 의존성은 아니다.

4. 실행 순서 (data/processed 는 .gitignore 라 클론 후 재생성 필요)
------------------------------------------------------------------------
  ./venv/bin/python keypoint_extractor.py    # raw -> landmarks.csv + skeleton.mp4
  ./venv/bin/python smooth_landmarks.py      # -> *_smoothed.csv
  ./venv/bin/python normalize_landmarks.py   # -> *_normalized.csv

이후 단계는 *_normalized.csv 를 입력으로 삼는다.


------------------------------------------------------------------------
5. 확정된 설계 결정 (⚠ 재논의하지 말 것)
------------------------------------------------------------------------
1) 각도 기반 비교 — 관절 각도는 크기·위치·해상도에 무관
   (scale/translation invariant). 그래서 기준 영상과 사용자 영상의
   해상도·FPS·프레임 수를 맞출 필요가 없다.

2) 정면·측면은 서로 비교하지 않는다 — 정면<->정면기준, 측면<->측면기준으로
   뷰별 독립 비교(DTW 도 뷰별로 따로). 따라서 정규화 스케일(상체 길이)이
   정면·측면 간 일치할 필요 없음.

3) 정규화 방식: norm = (joint - pelvis) / torso_scale
   - pelvis = midpoint(왼골반23, 오른골반24) -> 원점
   - torso_scale = 어깨중심(11,12)~골반중심(23,24) 거리의
     영상 전체 median (상수 1개)
     (프레임별로 쓰면 스쿼트 시 상체 기울기의 투영 변화가 스케일에 섞여 왜곡됨)
   - 종횡비 보정: x·너비, y·높이, z·너비로 픽셀 환산 후 계산.
     해상도는 video_resolution() 이 영상 첫 프레임에서 자동 감지 -> 하드코딩 금지.

4) 좌우 반전은 측면만 — 발끝(31,32)이 발뒤꿈치(29,30)보다 +x 쪽에 오도록 통일.
   정면은 좌우 대칭 평가를 해야 하므로 반전하지 않음.

5) DTW 의 역할 = "판정"이 아니라 "시간 정렬" — 기준과 사용자의 속도·프레임 수가
   달라도 같은 국면끼리 대응시켜 전체 유사도·템포 비교 + 최저점 매칭.
   구체 항목 판정은 6)처럼 최저점 프레임에서 임계값 비교가 더 정확하고 해석이 쉽다.
   (DTW + 임계값 판정 병행)

6) 판정 기준: 기본은 기준 영상 값 ± 허용범위(상대 비교), 일부 항목은 스쿼트
   바이오메카닉 절대 기준(예: 무릎 굽힘 <= 90도, 고관절 <= 무릎 높이=평행)을 섞는다.


------------------------------------------------------------------------
6. 남은 단계 (순서대로)
------------------------------------------------------------------------

[5단계] 특징(각도/비율) 추출   <- 다음에 할 것
입력: *_normalized.csv. 프레임별 특징 시계열을 뽑는다.
- 측면 "카메라쪽 다리 선택"은 별도 단계 없이 특징 함수 안에서 visibility(v)로
  처리하기로 함 (더 높은 visibility 의 좌/우 관절을 사용).
- 각도는 세 점으로 계산: angle(A,B,C) = B 에서 BA, BC 벡터의 사잇각.

측면 특징:
  무릎 굽힘 각도  = angle(고관절, 무릎, 발목)  -> 깊이 지표
  고관절 각도    = angle(어깨, 고관절, 무릎)
  몸통(척추)기울기 = (어깨-고관절) 벡터와 수직축 사이 각
  엉덩이 깊이     = 고관절 y vs 무릎 y (평행 이하인가)
  무릎 전방 이동  = 무릎 x vs 발끝(31/32) x
  정강이 각도    = (무릎-발목) 벡터와 수직축 사이 각

정면 특징:
  스탠스 너비      = 발목간 거리 / 어깨너비 (비율)
  무릎 모임(valgus) = 무릎간 거리 / 발목간 거리 (작아지면 안쪽으로 모임)
  좌우 대칭        = 좌·우 무릎 각도, 고관절 높이 차이

검증: 기준 영상에서 특징값을 뽑아 뼈대 영상과 대조. 예) 최저점 무릎 굽힘 각도가
      90도대인지, 스탠스 너비가 상식적인 값인지 확인.

[6단계] rep 최저점(bottom) 검출
- 하강 -> 최저 -> 상승 구간 분리. 최저점 = 측면 무릎 굽힘 각도 최소 또는
  골반 y 최대(화면 아래) 지점.
- 여러 rep 이 있으면 rep 별로 분할(현재 기준 영상은 1 rep 로 보임).
검증: 검출된 최저 프레임이 실제 딥 스쿼트 지점인지 눈으로 확인.

[7단계] DTW 정렬
- 기준 특징 시계열 vs 사용자 특징 시계열을 뷰별로 DTW 정렬.
- 라이브러리 후보: dtaidistance 또는 fastdtw (requirements 에 추가 필요).
- 정렬 경로로 "기준 최저점 <-> 사용자 최저점" 매칭.
검증: 정렬 경로가 국면을 맞게 잇는지, 최저점끼리 대응되는지.

[8단계] 임계값 판정 + 피드백 문구
- 각 항목을 최저점(및 필요한 구간)에서 기준값 ± 허용범위 / 절대기준과 비교
  -> 통과/실패.
- 실패 항목을 사람이 읽는 문구로 변환. 예:
    무릎간거리/발목간거리 < 0.85 -> "무릎이 안쪽으로 모입니다"
- (선택) 항목별 점수 -> 종합 스코어/리포트.
검증: 일부러 잘못된 자세 영상으로 기대한 피드백이 나오는지.


------------------------------------------------------------------------
7. 데이터 형식 참고
------------------------------------------------------------------------
- CSV 컬럼: frame_number, x0,y0,z0,v0, ..., x32,y32,z32,v32
           (총 1 + 33x4 = 133)
- 관절 번호: 11=왼어깨 12=오른어깨 23=왼골반 24=오른골반 25=왼무릎 26=오른무릎
            27=왼발목 28=오른발목 29=왼발뒤꿈치 30=오른발뒤꿈치
            31=왼발끝 32=오른발끝 0=코
- 정규화 CSV: x,y,z 는 정규화값(골반 원점, 상체길이=1),
             v(visibility)는 원본 유지.


------------------------------------------------------------------------
8. 사용자 영상 처리 시 주의
------------------------------------------------------------------------
- normalize_csv(csv, out, video_path, flip_side=...) 에 그 영상 경로만 넘기면
  해상도는 자동 감지. 측면이면 flip_side=True.
- 세로/가로 어떤 비율로 찍어도 종횡비 보정이 자동으로 맞는다.
